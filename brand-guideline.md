# KIAO 聰明學習 — Brand Guidelines

> 本文件為 KIAO 品牌視覺規範的結構化摘要，供 AI 在生成設計、素材、UI 時作為約束條件參考。

---

## 1. 品牌概述

- **品牌名稱**：KIAO 聰明學習
- **子品牌**：WORD UP
- **品牌定位**：以「聰明科技，有效學習」為核心，融合 AI 演算法與數據分析，提供個人化語言學習體驗
- **品牌調性**：科技感、理性、結構化、親和、高效

---

## 2. Logo 系統

### 2.1 組成元素

KIAO 的品牌識別由以下四個獨立視覺元素組成，可單獨使用或依規範組合：

| 元素名稱 | 內容 | 說明 |
|----------|------|------|
| Logo（標誌） | 幾何「K」字圖標 | 由四個幾何圖形（圓形 + 三角形 + 矩形）組成，品牌藍色 |
| English Wordmark | `KIAO` | 現代無襯線黑體字型 |
| Chinese Wordmark | `聰明學習` | 中文標準字 |
| Sub-Brand Wordmark | `WORD UP` | 子品牌標準字 |

### 2.2 Logo 組合模式（Logo Lockup）

以下是官方認可的組合方式。**請勿自行創建組合，僅使用以下列出的模式**：

#### 英文標誌組合
| 模式 | 排列方式 |
|------|----------|
| 英文橫式 | `[Logo] [KIAO]`（左右水平排列） |
| 英文直式 | `[Logo]` 上方 + `[KIAO]` 下方（垂直置中堆疊） |

#### 中文標誌組合
| 模式 | 排列方式 |
|------|----------|
| 中文橫式 | `[Logo] [聰明學習]`（左右水平排列） |
| 中文直式 | `[Logo]` 上方 + `[聰明學習]` 下方（垂直置中堆疊） |

#### 標準字組合（Wordmark Lockup，無 Logo 圖標）
| 模式 | 排列方式 |
|------|----------|
| 直式 | `KIAO` 上方 + `聰明學習` 下方 |
| 橫式 | `KIAO 聰明學習`（左右並排） |

#### 雙語標誌組合（Bilingual Logo Lockup）
| 模式 | 排列方式 |
|------|----------|
| 直式 A | `[Logo] + KIAO`（上方）→ `聰明學習`（下方） |
| 直式 B | `[Logo] + 聰明學習`（上方）→ `KIAO`（下方） |
| 橫式 | `[Logo] + KIAO + 聰明學習`（全部水平排列） |

#### 子品牌標誌組合
| 模式 | 排列方式 |
|------|----------|
| 橫式 | `[Logo] [WORD UP]`（左右水平排列） |
| 直式 | `[Logo]` 上方 + `[WORD UP]` 下方 |

### 2.3 淨空範圍（Clear Space）

淨空範圍是 Logo 周圍必須保留的最小空白區域，此區域內不可放置任何其他圖形或文字元素。

| 視覺元素 | 淨空單位 | 單位定義 |
|----------|----------|----------|
| Logo 標誌（單獨） | 四周各留 `1/2 K` | K = Logo 圖標的完整高度 |
| KIAO Wordmark | 四周各留 `O` | O = Wordmark 中字母「O」的寬度 |
| 聰明學習 Wordmark | 四周各留 `X` | X = 中文字「聰」的高度 |
| WORD UP Wordmark | 四周各留 `O` | O = Wordmark 中字母「O」的寬度 |
| 英文標誌組合（橫式/直式） | 四周各留 `1/2 K` | K = 組合中 Logo 圖標的高度 |
| 中文標誌組合（橫式/直式） | 四周各留 `1/2 K` | 同上 |
| 雙語標誌組合 | 四周各留 `K` | K = 組合中 Logo 圖標的高度 |
| 子品牌橫式組合 | 四周各留 `K` | 同上 |
| 子品牌直式組合 | 上下左右各留 `1/2 K` | 同上 |

### 2.5 禁止使用方式

以下任何一項都不允許，適用於所有視覺元素（含 Logo、Wordmark、組合，以及 WORD UP）：

| 類別 | 禁止行為 |
|------|----------|
| 變形 | 不可拉伸、壓縮、旋轉、傾斜、扭曲 Logo 或 Wordmark |
| 文字置換 | 不可替換 Wordmark 中的字型或文字內容（例如：不可將中文標誌組合中的「聰明學習」替換為其他文字） |
| 裝飾效果 | 不可添加陰影（drop shadow）、外框線（stroke/border）、光暈、3D 效果 |
| 色彩修改 | 不可使用非規範色彩、不可淡化（降低透明度）、不可填入材質或圖案 |
| 結構改造 | 不可新增圖形元素、不可移除組成部分、不可改變 Logo 各元素的相對位置與比例 |

### 2.6 Logo 色彩模式

根據使用背景，選擇正確的色彩模式：

| 優先順序 | 背景條件 | Logo 圖標顏色 | Wordmark 顏色 | 適用場景 |
|----------|----------|-------------|--------------|---------|
| A（最優先） | 白色或淺色背景 | 品牌藍 `content-brand` | 黑色 `#000000` | 官方文件、網站淺色主題、行銷素材 |
| B（輔助） | 白色或淺色背景 | 黑色 `#000000` | 黑色 `#000000` | 色彩受限的印刷（如單色列印） |
| C（輔助） | 品牌藍 `content-brand` 背景 | 白色 `#FFFFFF` | 白色 `#FFFFFF` | 品牌主視覺、深藍色背景 |
| D（輔助） | 白色或淺色背景 | 黑色 `#000000` | 黑色 `#2A2B2D` | 黑白印刷 |
| E（輔助） | 黑色背景 | 白色 `#FFFFFF` | 白色 `#FFFFFF` | 深色主題、暗色模式 |

**選擇邏輯**：優先使用模式 A（彩色）。僅在色彩數量受限、深色背景、或特殊印刷條件下使用 B–E。

## 3. 字型系統（Typography）

### 3.1 字型家族

| 語言 | 字型名稱 | 用途 |
|------|---------|------|
| 英文 | **Figtree** | 所有英文文字 |
| 中文 | **Noto Sans TC** | 所有中文文字 |

### 3.2 文字層級規格

> **術語定義**：
> - `letter-spacing`：字母之間的間距（CSS `letter-spacing`）
> - `line-height`：行高（CSS `line-height`，以百分比表示時為相對於 `font-size` 的倍率）

#### 展示與標題文字、內文與輔助文字、分類區塊文字（Segmentation）

> 直接參考 Design system file 裡的 text style，勿自行新增: https://www.figma.com/design/cM4wqQ8BuMgp6WU1i09EGO/-Lab--Design-System?node-id=141-7386&p=f&t=ibm9DKqcu6pwayGR-0

### 3.3 字型使用注意事項

- **最小可讀字級**：一般正文建議 14px。10px–12px 僅限備註或非關鍵資訊使用。
- **desktop 與 mobile screen 差異**：請參考 Design system file 裡的 text style token 設定。
- **Display 層級的特殊處理**：Display 層級字型間距僅針對 letter-spacing 設定，Noto Sans TC 與 Figtree 共用相同 letter-spacing 值。

## 4. 網格系統（Grid System）

- 基本上採用 **4-pixel grid**，所有間距與尺寸值為 4 的倍數
- 常用間距值序列：`0, 4, 8, 12, 16, 20, 24, 28, 32, 36, 40, 44, 48, 52, 56, 60, 64, 68, 72`
- **例外情況**：為了視覺協調的情況，可使用 6px 或 10px

## 5. 色彩系統（Color）

### 5.1 品牌核心色

| 角色 | Token 名稱 | HEX | RGB | CMYK | 使用場景 |
|------|-----------|-----|-----|------|---------|
| 背景白 | white | `background-primary` | 頁面背景、卡片背景 |
| 品牌主色 | `content-brand` | Logo 標誌、主要按鈕、重點強調、連結 |
| 品牌次色 | `content-accent` | Highlight 狀態、次要強調、促銷標籤 |
| 主要文字色 | `content-primary` | 標題文字、正文文字 |
| 強調橘 | `content-accent-strong` | 警告、緊急狀態、促銷價格 |
| 淺灰背景 | `background-secondary` | 區塊背景、分隔用底色 |

### 5.2 色彩使用規則

1. **品牌色面積比例**：品牌藍與品牌橘的使用面積不超過整體版面的 25%
2. **文字顏色**：正文與標題使用 `content-primary`（`#2A2B2D`）。次要資訊文字可使用較淺灰色（如 `content-secondary`）
3. **無障礙對比**：每個色彩均已標注 WCAG 對比度等級（見下方色票表），選色時請預設確保文字與背景的對比度至少達到 AA 標準

## 6. 漸層色（Gradience）

### 6.1 品牌漸層色

品牌漸層以藍色系列為主。可用於視覺插圖設計、背景、圖形，以及純 Logo 展示時的背景。

| 漸層名稱 | 色彩組成 | CSS 參考寫法 |
|----------|---------|-------------|
| 品牌藍漸層 | blue-lighter-400 → blue-lighter-50 → blue-lighter-500 | `linear-gradient(135deg, #5E88F7 0%, #EBF0FE 50%, #366AF5 100%)` |

### 6.2 品牌混合漸層（Brand Gradience Mix）

將藍色系列與橘色系列混合，適用於品牌主視覺與行銷素材。

| 風格 | 描述 | 色彩範圍 |
|------|------|---------|
| 暖調混合 | 淺藍 → 暖橘 → 淺白 | blue-lighter-400 / orange-400 / white |
| 冷調混合 | 深藍 → 暖橘 → 深藍 | blue-900 / orange-400 / blue-700 |

### 6.3 UI 漸層應用

| UI 元素 | 填充漸層（Color-Filled） | 描邊漸層（Color-Border） |
|---------|----------------------|----------------------|
| Loading Bar | `blue-500` → `blue-lighter-400` | `blue-lighter-50` → `blue-lighter-400` |
| Loading Bar（含橘色） | `blue-500` → `blue-lighter-400` | `orange-500` → `orange-300` |
| Segmentation 標記 | `blue-500` 實色（無漸層） | — |
| Highlight Content / List | `grayscale-50` 或 `white` 背景 | — |

## 7. 品牌圖形（Brand Pattern）

### 7.1 設計原則

- 品牌圖形由 **Logo Icon 的幾何形狀**（圓形、三角形、矩形）放大、裁切、重組而成
- 圖形為抽象裝飾用途，不承載具體語義

### 7.2 配色規則

| 背景主色 | 圖形表現 | 使用場景 |
|----------|---------|---------|
| 品牌藍（`content-brand` 或深藍漸層） | 淡藍色、白色、深藍色漸層的半透明圖形 | 預設主視覺 |
| 品牌橘（`content-accent` 或橘色漸層） | 淡橘色、白色圖形 | 分類或區分用途的第二設計 |

### 7.3 圖形豐富度技巧

- 主要圖形使用單一色系（藍或橘）的漸層
- 可將其中 **一個** 幾何圖形轉為另一色系（如藍底中加入一個橘色漸層圖形），以增加畫面的視覺對比

## 8. 插畫風格（Illustration）

### 8.1 風格定義

| 特徵 | 規範 |
|------|------|
| 圖形語言 | 簡約幾何圖形（圓形、矩形、三角形、菱形） |
| 同時使用顏色數量 | 最多 **2 種色系**（例如：藍 + 橘，或藍 + 白） |
| 質感表現 | 以漸層維持質感與層次 |
| 科技感處理 | 產品相關插圖可使用 **GlassBlur 效果**（CSS: `backdrop-filter: blur(...)` 或類似毛玻璃效果） |

### 8.2 Iconographic 風格

| 特徵 | 規範 |
|------|------|
| 配色 | 以 **單色** 為主 |
| 表現方式 | 可使用實色填充，或單色漸層 |
| 風格 | 線性、幾何、簡約 |
